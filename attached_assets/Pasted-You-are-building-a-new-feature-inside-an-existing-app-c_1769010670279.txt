2. Database Schema (Drizzle)

Create the following tables using Drizzle ORM:

form_templates
- id (uuid, pk)
- name (text)
- type (enum: job_sheet | client_form | quote_sheet | signoff)
- status (enum: draft | published | archived)
- created_by (uuid)
- created_at (timestamp)
- updated_at (timestamp)

form_template_versions
- id (uuid, pk)
- template_id (uuid, fk)
- version (int)
- schema (jsonb)
- created_at (timestamp)
- published_at (timestamp, nullable)

form_submissions
- id (uuid, pk)
- template_version_id (uuid, fk)
- entity_type (enum: job | client | quote)
- entity_id (uuid)
- submitted_by (uuid)
- status (enum: draft | submitted)
- data (jsonb)
- created_at (timestamp)
- submitted_at (timestamp, nullable)

form_assets
- id (uuid, pk)
- submission_id (uuid, fk)
- field_key (text)
- asset_type (enum: photo | signature)
- file_path (text)
- created_at (timestamp)

IMPORTANT:
- Submissions MUST reference template_version_id, never template_id
- Old submissions must never change if templates are edited

---

3. Template Schema Format (JSON)

Each template version stores a JSON schema like:

{
  "name": "Job Sheet – Standard",
  "style": "clean",
  "fields": [
    {
      "type": "text",
      "key": "job_ref",
      "label": "Job Reference",
      "required": true,
      "prefill": "job.ref"
    },
    {
      "type": "textarea",
      "key": "work_done",
      "label": "Work Completed",
      "required": true
    },
    {
      "type": "photo",
      "key": "before_photos",
      "label": "Before Photos",
      "multiple": true
    },
    {
      "type": "signature",
      "key": "customer_signature",
      "label": "Customer Signature",
      "required": true
    }
  ]
}

Supported field types:
- text
- textarea
- number
- date
- select
- multiselect
- checkbox
- yesno
- photo
- signature
- repeatable_group

4. Backend (Express)

Create routes:

GET /forms/templates
GET /forms/templates/:id
POST /forms/templates
POST /forms/templates/:id/publish

GET /forms/templates/:versionId/schema

POST /forms/submissions
GET /forms/submissions
GET /forms/submissions/:id

POST /forms/assets/upload

Rules:
- Publishing creates a new immutable template_version
- Editing a template creates a new draft version
- Only published versions are fillable

Use Zod to validate:
- Template schema
- Submission payloads
