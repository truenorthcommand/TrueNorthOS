You are building a UK electrical certificate web app. Implement a multi-circuit EICR generator (NOT one-circuit). Use the existing stack:
Frontend: React + TypeScript + Vite + Tailwind + shadcn/ui + TanStack Query + Wouter
Backend: Node.js + Express + TypeScript
DB: PostgreSQL + Drizzle ORM
Validation: Zod
PDF: pdf-lib

GOAL
Create an MVP where a user can:
1) Create an EICR Report
2) Add one or more Distribution Boards (DBs)
3) Add multiple Circuits per DB (unlimited)
4) Enter test results per circuit (R1+R2, Zs, IR L-N, IR L-E, IR N-E, polarity)
5) Add Observations with codes C1/C2/C3/FI
6) Export a PDF EICR that includes:
   - Report front page summary (basic)
   - Observation list
   - Schedule of Circuit Details + Test Results
   - Pagination/continuation pages when circuits exceed page capacity (e.g. 10 circuits per page)
   - Page numbering “Page X of Y”
   - Per DB section headers
Do NOT limit to one circuit. Circuits are stored as rows in a circuits table and rendered with chunked pagination.

PROJECT STRUCTURE
Make a monorepo in one Replit project:
- /server (Express API, Drizzle, PDF generation)
- /client (React UI, forms, tables)
Use a single root package.json with workspaces or two package.json files; whichever is easier on Replit.

DATABASE SCHEMA (Drizzle)
Create these tables:
eicr_reports:
  id uuid pk
  reference text
  standard text (default "BS 7671:2018+A2:2022")
  outcome text nullable ("SATISFACTORY"|"UNSATISFACTORY")
  client_name text
  client_address text
  installation_address text
  created_at timestamp

distribution_boards:
  id uuid pk
  report_id uuid fk -> eicr_reports.id
  db_ref text (e.g. "DB1")
  location text

circuits:
  id uuid pk
  board_id uuid fk -> distribution_boards.id
  circuit_ref text (e.g. "1","2","L1")
  description text
  breaker_type text ("MCB"|"RCBO"|"OTHER")
  breaker_rating int (e.g. 6,16,32)
  curve text nullable ("B"|"C"|"D")
  rcd_protected boolean default false
  rcd_type text nullable ("AC"|"A"|"F"|"B")
  rcd_idelta_ma int nullable (e.g. 30)
  line_mm2 real nullable
  cpc_mm2 real nullable
  r1_plus_r2 real nullable
  zs real nullable
  ir_ln real nullable
  ir_le real nullable
  ir_ne real nullable
  polarity_ok boolean default true
  remarks text nullable
  sort_order int default 0

observations:
  id uuid pk
  report_id uuid fk
  code text ("C1"|"C2"|"C3"|"FI")
  location text
  circuit_ref text nullable
  text text
  created_at timestamp
